AWS Networking Project

Building Custom VPC Architectures for Secure and Scalable Cloud Environments

In this nine-part project series, I designed and built custom VPC architectures from scratch, incorporating public and private subnets, route tables, and internet gateways to create secure and scalable cloud environments.

To enhance network security and connectivity, I:

  • Launched EC2 instances in both public and private subnets, ensuring controlled access to resources.
  • Configured VPC Flow Logs to monitor network traffic, sending logs to CloudWatch for analysis using Logs Insights.
  • Set up VPC peering to establish a private connection between two VPCs, allowing secure communication without exposing traffic to the public internet.
  • Created an S3 Gateway VPC Endpoint, enabling direct and secure access to Amazon S3.

Services & Tools Used:

  • Networking & Security: Amazon VPC, VPC Flow Logs, VPC Peering
  • Compute Resources: Amazon EC2
  • Monitoring & Analytics: CloudWatch Logs & Insights
  • Connectivity & Integration: S3 Gateway VPC Endpoint
